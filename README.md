## How I think about systems

```txt
controllers should stay thin
business rules should be explicit
infrastructure should be replaceable
APIs should be boring, stable, and documented
packages should remove mess from applications, not add new mess
```

A good backend is not measured by how clever it looks.  
It is measured by how safely it can survive change.
